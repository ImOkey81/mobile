# MobileHost

Приложение на Jetpack Compose для удалённой диагностики сети через backend `91.107.126.43:8080`.
Основная цель — быстро проверить доступность сервисов и сетевую связность с мобильного устройства,
чтобы получить базовую картину состояния сети без доступа к отдельным утилитам на ПК.

Проект позволяет выполнять несколько типов проверок:
- **HTTP** — проверка доступности URL и получения ответа сервера.
- **Ping** — измерение задержки и доступности узла по ICMP.
- **TCP порт** — проверка доступности конкретного порта (например, 22/80/443).
- **Traceroute** — просмотр маршрута до хоста и промежуточных узлов.
- **DNS lookup** — проверка разрешения доменного имени в IP-адрес.

Результаты отображаются в интерфейсе и сохраняются в состоянии экрана, чтобы ими можно было
пользоваться при переключении вкладок или повороте экрана.

## Архитектура и технологии

Проект разделён по слоям MVVM и использует следующие ключевые технологии:
- **data** — `NetworkClient`, реализации репозиториев и работа с API.
- **domain** — модели, интерфейсы репозиториев и use case для авторизации и отправки проверок.
- **presentation** — UI-компоненты на Compose и `ViewModel` с состоянием на `StateFlow`.
- **di** — простой модуль для сборки зависимостей и фабрик ViewModel.

Состояние хранится во `ViewModel`, поэтому при повороте экрана вкладки, введённые значения и результаты запросов сохраняются. Навигация построена через `navigation-compose`: сначала отображается экран авторизации, после сохранения логина/пароля открывается экран проверок.

## Сборка и запуск

### Требования
- Android Studio (Hedgehog или новее) или Gradle CLI
- JDK 17

### Сборка APK
```bash
./gradlew :app:assembleDebug
```

### Запуск
- Откройте проект в Android Studio и запустите конфигурацию `app` на устройстве или эмуляторе.
- Либо установите собранный APK на устройство.

Запускать на девайсе/эмуляторе с интернетом. Сервер ожидает Basic Auth: логин и пароль задаются на первом экране приложения и используются для всех последующих запросов.

## Структура репозитория

- `app/src/main/java/...` — исходный код приложения (слои `data`, `domain`, `presentation`, `di`).
- `app/src/main/res` — ресурсы (иконки, строки, темы).
- `gradle/`, `build.gradle.kts`, `settings.gradle.kts` — настройки сборки.

## Используемые технологии

- Jetpack Compose Material 3
- Navigation Compose
- Kotlin coroutines + StateFlow для сохранения состояния
- OkHttp для HTTP-запросов
- Ручной DI модуль `AppModule`
